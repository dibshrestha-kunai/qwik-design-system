## Component API

### Root
The root component manages the radio group's state and behavior.
```tsx
<RadioGroup.Root
  value={value}           // Controlled value
  onChange$={(v) => {}}  // Change handler
  disabled={false}       // Disable entire group
  required={true}        // Make selection required
  orientation="vertical" // Layout orientation
>
  {/* Radio items */}
</RadioGroup.Root>
```

### Item
Container for individual radio options.
```tsx
<RadioGroup.Item value="option1">
  <RadioGroup.Label>Option 1</RadioGroup.Label>
  <RadioGroup.Trigger value="option1" _index={0}>
    <RadioGroup.Indicator value="option1" />
  </RadioGroup.Trigger>
</RadioGroup.Item>
```

### Trigger
The interactive button component that handles selection.
```tsx
<RadioGroup.Trigger
  value="option1"  // Must match parent Item value
  _index={0}       // For keyboard navigation order
  disabled={false} // Optional individual disable
/>
```

### Indicator
Visual indicator for the selected state.
```tsx
<RadioGroup.Indicator value="option1">
  {/* Custom indicator content */}
</RadioGroup.Indicator>
```

## Keyboard Navigation

The component supports full keyboard navigation:
-	Tab: Focus the radio group
-	Space/Enter: Select focused option
-	Arrow keys: Navigate between options (respects orientation)
-	Home/End: Jump to first/last option

## Accessibility

Built following the WAI-ARIA radio group pattern, the component includes:
-	Proper role attributes (radiogroup, radio)
-	Aria states (checked, disabled)
-	Focus management
-	Keyboard navigation
-	Form integration
-	Error states
-	Description support

## Form Integration

The component provides several features for form integration:
```tsx
<RadioGroup.Root
  name="options"        // Form field name
  required={true}      // Required validation
  form="form-id"       // Form connection
>
  {/* Radio items */}
  <RadioGroup.ErrorMessage>
    Please select an option
  </RadioGroup.ErrorMessage>
</RadioGroup.Root>
```
